Story

IN THE METAVERSE, WHETHER YOU HAVE MOBILITY IMPAIRMENTS OR NOT IS NOT GOING TO AFFECT HOW PEOPLE PERCEIVE YOU

The metaverse idea I propose here aims to revolutionize the lives of people with mobility impairments. This would be the most unique experience that would happen in the life of people with mobility impairments. As in video games, whether you have mobility impairments or not, it will not affect drastically your gaming experience. Yet, gaming is not real life as it lacks movements. Metaverse can bring real "normal" life to people with mobility impairments. This super new unique idea is currently not useful at all because it's so hard to develop, but yet it is so unique and I hope future makers will make it possible.

The Contest Masters told me in their feedback that " THE POSSIBILITIES ARE ENDLESS " with my Metaverse idea. So it's really really NEW idea.

The issue of mobility impairments in the metaverse highlights a fundamental problem: existing implementations inadequately cater to the needs of individuals with such impairments. Current platforms often rely on extensive physical movements for virtual actions, presenting obstacles for those with mobility impairments . To address this, adapting metaverse interfaces to allow minimal physical movements for virtual interactions is crucial . However, despite promising strategies, challenges persist due to technological limitations and design complexities. Further research and innovation are necessary to achieve comprehensive accessibility, ensuring the metaverse truly becomes an inclusive platform for all users.

My unique solution to address the challenges faced by individuals with mobility impairments in the metaverse is the development of advanced interface technologies. These technologies would allow for the translation of minimal physical movements into meaningful virtual actions, thereby reducing the reliance on extensive physical mobility within the virtual environment. By incorporating such interfaces, individuals with mobility impairments would be able to navigate and interact within the metaverse more effectively, promoting greater accessibility and inclusivity.

I think the hardware from the sponsors are super amazing to support this idea, here is how they support novelty :

Blues Kit :

  • Swan v3 : Serves as the central microcontroller, integrating various input devices and sensors to facilitate minimal physical movement detection.
  • Notecarrier AL or AA : Provides essential power and USB connectivity for the Swan v3, ensuring stable operation and easy integration with other hardware components.
  • Notecard Cellular NBGL : Enables reliable data transmission over cellular networks, allowing users to interact with the metaverse from anywhere with cellular coverage, especially beneficial for users in remote areas.
  • Notecard WiFi v1 : Offers an alternative to cellular connectivity, using WiFi for data transmission, ensuring flexibility and adaptability in different environments

Nordic Semiconductor's nRF52840 DK :

  • Provides robust Bluetooth connectivity, facilitating seamless communication between various input devices (e.g., motion sensors, adaptive controllers) and the metaverse interface, ensuring real-time interaction with minimal delay

Seeed Studio :

  • XIAO ESP32S3 Sense : Utilizes its motion detection and sound microphone capabilities to capture minimal physical movements and voice commands from users. The built-in AI can process these inputs to generate corresponding actions in the metaverse. Its WiFi and Bluetooth features ensure flexible connectivity.
  • XIAO Grove Shield : Enhances the XIAO ESP32S3 with additional battery life, memory, and connectors, supporting extended usage and connectivity options for various sensors and input devices, ensuring the system remains operational and responsive.

import pygame
import sys

# Constants
WIDTH, HEIGHT = 800, 600
WHITE = (255, 255, 255)
BLACK = (0, 0, 0)
BLUE = (0, 0, 255)
FPS = 30

# User data
users = {"admin": "password123"}

# Initialize Pygame
pygame.init()
screen = pygame.display.set_mode((WIDTH, HEIGHT))
pygame.display.set_caption("Metaverse Mobility")
clock = pygame.time.Clock()

# Load assets
font = pygame.font.Font(None, 36)

# Authentication
def login():
    user_input = ""
    password_input = ""
    user_box = pygame.Rect(100, 100, 140, 32)
    password_box = pygame.Rect(100, 150, 140, 32)
    active_user = False
    active_password = False
    color_inactive = pygame.Color('lightskyblue3')
    color_active = pygame.Color('dodgerblue2')
    color_user = color_inactive
    color_password = color_inactive
    logged_in = False
    
    while not logged_in:
        for event in pygame.event.get():
            if event.type == pygame.QUIT:
                pygame.quit()
                sys.exit()
            if event.type == pygame.MOUSEBUTTONDOWN:
                if user_box.collidepoint(event.pos):
                    active_user = True
                    active_password = False
                elif password_box.collidepoint(event.pos):
                    active_password = True
                    active_user = False
                else:
                    active_user = False
                    active_password = False
                color_user = color_active if active_user else color_inactive
                color_password = color_active if active_password else color_inactive
            if event.type == pygame.KEYDOWN:
                if active_user:
                    if event.key == pygame.K_RETURN:
                        active_user = False
                        color_user = color_inactive
                    elif event.key == pygame.K_BACKSPACE:
                        user_input = user_input[:-1]
                    else:
                        user_input += event.unicode
                elif active_password:
                    if event.key == pygame.K_RETURN:
                        active_password = False
                        color_password = color_inactive
                    elif event.key == pygame.K_BACKSPACE:
                        password_input = password_input[:-1]
                    else:
                        password_input += event.unicode
        
        screen.fill((30, 30, 30))
        user_surface = font.render(user_input, True, WHITE)
        password_surface = font.render('*' * len(password_input), True, WHITE)
        screen.blit(user_surface, (user_box.x + 5, user_box.y + 5))
        screen.blit(password_surface, (password_box.x + 5, password_box.y + 5))
        pygame.draw.rect(screen, color_user, user_box, 2)
        pygame.draw.rect(screen, color_password, password_box, 2)
        
        pygame.display.flip()
        clock.tick(FPS)
        
        if not active_user and not active_password:
            if user_input in users and users[user_input] == password_input:
                logged_in = True
                print("Login successful!")
            else:
                user_input = ""
                password_input = ""
                print("Invalid credentials, try again.")
                
    return True

# Function to translate minimal movements into bigger actions
def amplify_movement(dx, dy, scale=5):
    return dx * scale, dy * scale

# VR Environment
def vr_environment():
    player_pos = [WIDTH // 2, HEIGHT // 2]
    base_move_step = 1
    scale_factor = 5  # Scale factor to amplify movements
    
    running = True
    while running:
        dx, dy = 0, 0
        for event in pygame.event.get():
            if event.type == pygame.QUIT:
                running = False
        
        keys = pygame.key.get_pressed()
        if keys[pygame.K_LEFT] or keys[pygame.K_a]:
            dx = -base_move_step
        if keys[pygame.K_RIGHT] or keys[pygame.K_d]:
            dx = base_move_step
        if keys[pygame.K_UP] or keys[pygame.K_w]:
            dy = -base_move_step
        if keys[pygame.K_DOWN] or keys[pygame.K_s]:
            dy = base_move_step
        
        # Amplify the movement
        dx, dy = amplify_movement(dx, dy, scale_factor)
        player_pos[0] += dx
        player_pos[1] += dy
        
        # Ensure player stays within bounds
        player_pos[0] = max(0, min(WIDTH, player_pos[0]))
        player_pos[1] = max(0, min(HEIGHT, player_pos[1]))
        
        screen.fill(BLACK)
        pygame.draw.circle(screen, BLUE, player_pos, 20)
        
        pygame.display.flip()
        clock.tick(FPS)

# Main function
def main():
    if login():
        vr_environment()
    pygame.quit()

if __name__ == "__main__":
    main()
